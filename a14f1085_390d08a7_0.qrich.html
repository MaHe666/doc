首先将五个User对象全部放到List中

之前也说了，集合就用来存储

List<User> list = Arrays.asList(u1,u2,u3,u4,u5);


调用list对象的stream()方法

list.stream();

可以发现，他这里返回值类型是Stream<E>，说明他在处理的时候就已经拿到了集合保存对象的具体类型了。而这里的类型为User

刚才说了，集合用来存储，而计算要交给Stream来进行流式计算

上面的stream()方法返回的就是Stream对象。而我们需要调用Stream对象的filter对象进行过滤

可以看到这个方法的参数是一个Predicate 断定型接口实例

所以我们这里的代码，在搭配上lambda表达式的情况下需要这样写来处理id为偶数的用户

list.stream().filter(u -> u.getId() % 2 == 0);

再使用forEach方法去循环遍历

list.stream().filter(u -> u.getId() % 2 == 0).forEach(System.out::println);


然后我们再加入年龄大于23的条件

list.stream()

.filter(u -> u.getId() % 2 == 0)

.filter(u -> u.getAge() > 23)

.forEach(System.out::println);


然后我们看一下第三个条件：用户名转为大写字母。这里用到的就是Function函数型接口，因为需要输出处理后的输入值

所以这里的代码应该这样写

list.stream()

.filter(u -> u.getId() % 2 == 0)

.filter(u -> u.getAge() > 23)

.map(u -> u.getName().toUpperCase())

.forEach(System.out::println);


接下来是用户名名字的倒序排序

这里用到的方法是sorted()

没有参数的sorted()方法是正常的正序排序(升序)，所以咱们想要倒序则需要调用以下方法

ps: 使用Comparator接口的compare方法也可以实现

用第二个去比较第一个就好

代码

.sorted( (user1,user2) -> { return user2.compareTo(user1); })

代码：

list.stream()

.filter(u -> u.getId() % 2 == 0)

.filter(u -> u.getAge() > 23)

.map(u -> u.getName().toUpperCase())

.sorted(Comparator.reverseOrder())

.forEach(System.out::println);

然后再通过limit()方法，限定输出1个结果

最后的代码：

list.stream()

.filter(u -> u.getId() % 2 == 0)

.filter(u -> u.getAge() > 23)

.map(u -> u.getName().toUpperCase())

.sorted(Comparator.reverseOrder())

.limit(1)

.forEach(System.out::println);
