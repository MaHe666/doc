对于方法参数，Java编译器使用两个其他语言特性确定目标类型：重载解析和类型参数推断


考虑以下两个函数接口（java.lang.Runnable 和 java.util.concurrent.Callable<V>）：

public interface Runnable {

void run();

}


public interface Callable<V> {

V call();

}


方法Runnable。run不返回值，而Callable＜V＞。而call方法会。


假设您已按如下方式重载方法调用：

void invoke(Runnable r) {

r.run();

}


<T> T invoke(Callable<T> c) {

return c.call();

}


以下语句将调用哪个方法？

String s = invoke(() -> "done");


方法调用（Callable<T>）将被调用，因为该方法返回一个值；方法invoke（Runnable）没有。在这种情况下，lambda表达式（）->“done”的类型是Callable<T>。
