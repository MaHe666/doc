当sentinel重新启动的时候，sentinel-dashboard 中原来的数据将会全部消失，这样就需要重新定义限流规则，无疑是不可取的。sentinel默认是把配置放在内存里的


我们可以将sentinel中定义的限流规则保存到 Nacos配置中心 里面，实现持久化


1，添加sentinel对nacos的依赖支持

<dependency>

<groupId>com.alibaba.csp</groupId>

<artifactId>sentinel-datasource-nacos</artifactId>

</dependency>




2，在nacos中编写配置

注意！是Json格式


配置示例：

[

{

"resource":"sentinel-config",

"limitApp":"default",

"grade":1,

"count":2,

"strategy":0,

"controlBehavior":0,

"clusterMode":false

}

]




3，在项目中配置

spring:

cloud:

sentinel:

datasource:

ds:

nacos:

#naocs链接地址

server-addr: localhost:8848

#nacos链接的分组

group-id: DEFAULT_GROUP

#流控规则，rule-type配置表示该数据源中的规则属于哪种类型的规则(flow，degrade，authority，system，param-flow ，gw-flow , gw-api-group)

rule-type: flow

#读取配置文件的DataId

data-id: sentinel-config

#读取的配置文件类型为Json

data-type: json
