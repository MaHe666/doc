方法用于定义该类或该类实例的行为特征和功能实现。方法是类和对象行为特征的抽象。方法很类似于面向过程中的函数。面向过程中，函数是最基本单位，整个程序由一个个函数调用组成。面向对象中，整个程序的基本单位是类，方法是从属于类和对象的。


方法定义格式：

【修饰符】 返回值类型 方法名(形参){

// 方法体

}


方法声明中唯一必需的元素是方法的返回类型、名称、一对括号（）和大括号之间的主体｛｝。


更一般地说，方法声明有六个组件

  • 修饰符，如public、private和稍后将了解的其他修饰符。
  • 返回类型是方法返回值的数据类型，如果方法不返回值，则返回void。
  • 方法名字段名规则也适用于方法名，但约定有点不同。
  • 括号中的参数列表——一个逗号分隔的输入参数列表，前面是它们的数据类型，用括号（）括起来。如果没有参数，则必须使用空括号。
  • 稍后将讨论例外的list。
  • 方法体（括在方法代码的大括号之间，包括局部变量的声明）位于此处。


尽管方法名可以是任何合法标识符，但代码约定限制方法名。按照惯例，方法名称应为小写动词或以小写动词开头的多单词名称，后跟形容词、名词等。在多单词名称中，第二个和后续单词的第一个字母应大写。


通常，方法在其类中具有唯一的名称。然而，由于方法重载，一个方法可能与其他方法同名。


不能声明多个具有相同名称、相同数量和类型参数的方法，因为编译器无法区分它们。

编译器在区分方法时不考虑返回类型，因此即使两个方法具有不同的返回类型，也不能声明具有相同签名的两个方法。
