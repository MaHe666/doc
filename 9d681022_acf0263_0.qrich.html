如果类或其超类之一实现了可克隆接口，则可以使用clone（）方法从现有对象创建副本。要创建克隆，请编写

aCloneableObject.clone();


此方法的对象实现检查以查看在其上调用clone()的对象是否实现了Cloneable接口。如果该对象不支持，则该方法引发CloneNotSupportdException异常。异常处理将在后面的课程中介绍。

目前，您需要知道 clone() 必须声明为或者如果要编写一个 clone() 方法来覆盖Object中的方法。

protected Object clone() throws CloneNotSupportedException

或者

public Object clone() throws CloneNotSupportedException


如果调用clone()的对象确实实现了Cloneable接口，Object的clone()方法的实现会创建一个与原始对象相同类别的对象，并初始化新对象的成员变量，使其与原始对象的相应成员变量具有相同的值。


对于某些类，Object的 clone() 方法的默认行为工作得很好。但是，如果对象包含对外部对象的引用，比如ObjExternal，则可能需要重写 clone() 以获得正确的行为。否则，一个对象对ObjExternal所做的更改也将在其克隆中可见。这意味着原始对象及其克隆不是独立的 —— 要将它们分离，必须重写 clone()，以便它克隆对象和ObjExternal。然后，原始对象引用ObjExternal，克隆引用ObjExternal的克隆，因此对象及其克隆真正独立。
