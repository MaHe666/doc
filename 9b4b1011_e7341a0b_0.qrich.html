包含批注类型必须具有数组类型的值元素。数组类型的组件类型必须是可重复的注释类型。包含注释类型的Schedules的声明如下

public @interface Schedules {

Schedule[] value();

}


检索注解

在反射API中，有几种方法可以用来检索注释。返回单一注释的方法，如AnnotatedElement.getAnnotation(Class<T>)，其行为是不变的，因为它们只在有一个所请求类型的注释时才返回单一注释。如果有一个以上的请求类型的注释，你可以通过首先获得它们的容器注释来获得它们。通过这种方式，遗留的代码继续工作。在Java SE 8中引入了其他方法，通过容器注释扫描，一次返回多个注释，如AnnotatedElement.getAnnotationsByType(Class<T>)。


设计考量

设计注释类型时，必须考虑该类型注释的基数。现在可以零次、一次或多次使用注释，如果注释的类型标记为@Repeatable。还可以通过使用@Target元注释来限制可以使用注释类型的位置。例如，您可以创建只能用于方法和字段的可重复注释类型。仔细设计注释类型非常重要，以确保使用注释的程序员发现它尽可能灵活和强大。
