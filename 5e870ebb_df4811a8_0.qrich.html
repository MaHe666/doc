当你定义一个新的接口时，你就是在定义一个新的引用数据类型。

您可以在任何可以使用其他任何数据类型名称的地方使用接口名称。

如果你定义了一个类型为接口的引用变量，那么你分配给它的任何对象都必须是实现该接口的类的实例


例如，对于从实现Relatable的类实例化的任何对象，这里有一个查找一对对象中最大对象的方法

public Object findLargest(Object object1, Object object2) {

Relatable obj1 = (Relatable)object1;

Relatable obj2 = (Relatable)object2;

if ((obj1).isLargerThan(obj2) > 0)

return object1;

else

return object2;

}


通过将object1转换为Relatable类型，它可以调用isLargerThan方法。


如果要在各种各样的类中实现Relatable，那么可以将从这些类中实例化的对象与findLargest（）方法进行比较，前提是这两个对象都属于同一类。同样，它们都可以与以下方法进行比较

public Object findSmallest(Object object1, Object object2) {

Relatable obj1 = (Relatable)object1;

Relatable obj2 = (Relatable)object2;

if ((obj1).isLargerThan(obj2) < 0)

return object1;

else

return object2;

}


public boolean isEqual(Object object1, Object object2) {

Relatable obj1 = (Relatable)object1;

Relatable obj2 = (Relatable)object2;

if ( (obj1).isLargerThan(obj2) == 0)

return true;

else

return false;

}


这些方法适用于任何“Relatable”对象，无论它们的类继承是什么。当它们实现Relatable时，它们既可以是自己的类（或超类）类型，也可以是Relatable类型。这为它们提供了多重继承的一些优点，它们可以同时从超类和接口获得行为。
