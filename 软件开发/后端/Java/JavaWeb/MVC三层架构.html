                          早些年的架构

                          用户直接访问控制层，控制层就可以直接操作数据库：

                          Servlet -- JDBC --> 数据库

                          弊端：程序十分臃肿，不利于维护

                          Servlet的代码中：处理请求，响应，视图跳转，处理JDBC，处理业务代码，处理逻辑代码

                          架构

                          没有什么是加一层解决不了的

                          现在的三层架构

                          Model

                          业务处理：业务逻辑（Service）
                          数据持久层：CRUD （Dao）

                          View

                          展示数据
                          提供链接发起Servlet请求（比如a标签或者form的提交等等）

                          Controller

                          接收用户的请求：（req：请求参数，session信息等等）
                          交给业务层处理对应的代码
                          控制视图的跳转
