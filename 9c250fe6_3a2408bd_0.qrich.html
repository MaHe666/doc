使用lambda表达式创建匿名方法。然而，有时lambda表达式只会调用现有方法。在这些情况下，按名称引用现有方法通常更清晰。方法引用使您能够做到这一点；对于已经有名称的方法，它们是紧凑、易于读取的lambda表达式


看一下Comparator的代码示例：Comparator


我们知道了嵌套类，也知道了lambda表达式，我们可以对其进行优化

注意，接口Comparator是一个功能接口。因此，您可以使用lambda表达式，而不是定义并创建实现Comparator的类的新实例

将排序语句使用lambda表达式进行优化

Arrays.sort(arr, (o1, o2) -> o1.getNum() - o2.getNum());


我们还可以在lambda表达式的主体中调用方法

我们在Example类中添加如下方法

public static int compareByNum(Example o1, Example o2){

return o1.getNum() - o2.getNum();

}


然后修改排序的语句

Arrays.sort(arr, (a, b) -> Example.compareByNum(a, b));


由于此lambda表达式调用现有方法，因此可以使用方法引用而不是lambda表达式

Arrays.sort(arr, Example::compareByNum);



以上的 方法引用Example::compareByNum 和 lambda表达式 (a, b) -> Example.compareByNum(a, b) 在语义上相同，他们都具有以下特征

  • 它的正式参数列表是从Comparator<Example>复制的。比较，即（Example，Example）。
  • 它的主体方法调用的是Example.compareByNum(Example, Example)
