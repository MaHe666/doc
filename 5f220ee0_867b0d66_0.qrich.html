我们先简单声明一个字符串：

String string = "HelloWorld";


string变量的类型是一个String

String是Object类的子类。因此，String对象也是一个Object对象，他可以调用一个Object对象能有的操作

相反的情况不一定如此，Object对象不一定是一个String对象

类型转换体现在继承(extends)和实现(implements)方面，使用一种类型的对象替代另一种类型。

看一下以下语句：

Object obj = "HelloWorld";

则obj既是Object也是String（直到obj变量分配了一个不是String类型的对象）。这称为隐式转换

如果我们写:

String s = obj;

会出现一个编译时错误，因为编译器不知道obj是String

但是我们可以告诉编译器，我们承诺通过显示转换将String分配给obj

String s = (String) obj;

此强制转换将插入一个运行时检查，检查obj是否分配了一个String，以便编译器可以安全地假设obj是一个String。

如果obj在运行时不是String，就会引发异常


-----------------------------------------------------------------------


可以使用instanceof运算符对特定对象的类型进行逻辑测试。这可以避免由于不正确的强制转换而导致的运行时错误。例如：

if (obj instanceof String) {

String s = (String) obj;

}

这里，instanceof运算符验证obj是否引用了String，这样我们就可以在知道不会引发运行时异常的情况下进行强制转换。
