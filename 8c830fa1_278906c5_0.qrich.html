@Select("select * from `user` where id = #{id}")

User getUserById(@Param("id") Integer id);




在mapper文件中不写@Param，需要在xml中标明parameterType（也可以不用写，mybatis自动识别）


若在mapper文件中写上@Param，则在xml中可以不用写


通常在mapper中标明，在传入多个参数的时候具有扩展性，因为parameterType只允许传入一种类型


只有一个参数的时候可以不写@Param("")注解

例：

@Select("select * from `user` where id = #{id}")

User getUserByParam( Integer id);


但是当方法存在多个参数的时候，所有的基本类型的参数都必须在前面写上注解，这是规范

Map和其他引用对象（比如实体类对象）写注解的话传不进去


此外，@Param("") 括号里面的值可以和形参名不一致，相当于取别名

例：

@Select("select * from `user` where id = #{cid}")

User getUserByParam(@Param("cid") Integer id);
